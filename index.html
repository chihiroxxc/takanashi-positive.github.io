树（Tree）

树是一个被广泛运用的数据结构，用根结点和子节点来表示一个树。每个子节点的结构也遵循树的定义，所以很多树的问题都可以用递归来解决。

首先给出树的定义

class TreeNode{
    int val;
    TreeNode[] children;
}

深度优先搜索（DFS）

void dfs(TreeNode root){
    if(root==null)  return;
    visit(root);
    for(int i=0; i < root.children.length;i++){
        dfs(root.children[i]);
    }
}

广度优先搜索 (BFS) 层序遍历

public void levelOrder(TreeNode root) {
    if(root==null) return;
    //用辅助队列实现广度优先
    Queue<TreeNode> queue = new LinkedList();
    queue.offer(root);
    while(!queue.isEmpty()){
        int size = queue.size(); //该层总共的节点数
        for(int i = 0;i < size;i++){
            TreeNode node = queue.poll();
            visit(node); // 访问节点
            // 将该节点的子节点按照从左到右的顺序加入队列
            for(int j = 0; j < node.children.length; j++){
                queue.offer(node.children[j];
            }
        }
    }
}

LeetCode No.75 Sort Colors

发表于 2016-08-17   |   分类于 LeetCode   |  

Given an array with n objects colored red, white or blue, sort them so that objects of the same color are adjacent, with the colors in the order red, white and blue.

Here, we will use the integers 0, 1, and 2 to represent the color red, white, and blue respectively.

Could you come up with an one-pass algorithm using only constant space?

一列数字，只包括0，1，2三种，要求在不用比较排序算法的情况下排列成0..01..12..2这种形式，而且只能遍历一边数组（所以不能用统计排序的方法）。

这时可以想到一种非常简单的策略，遇到0的话把0放在数组的前面，遇到2的话把2放在数组后面，中间的1不做处理。用 end0来标识0序列结尾后的第一个位置，start2来标识2序列开始之前的一个位置。

从海量数据里选取k个数的相关问题

发表于 2016-04-14   |  

最近看到了从海量数据里选择 k 个数这类问题，如何减少时间复杂度和空间复杂度。首先看这类问题的比较简单的变种。

从海量数据里选取 Top k个数

  • 最朴素的想法：将海量数据排序，选取 Top k，最少的时间复杂度也是O(logn)
  • 改进版的想法：维护一个长度为k的有序数组，取海量数据的前k个初始化并排序，在遍历剩余数据时，若num_i大于于有序数组之最小值，则将其插入至有序数组中。这种算法的时间复杂度是O(n*10)
  • 改进的改进版：将长度为k的有序数组用小顶堆来代替，当num_i大于堆顶元素时，将其替换，对堆进行一次MIN_HEAPIFY操作。这种算法的时间复杂度是O(nlog10)

修改xib无效，运行后修改没有显示的问题

发表于 2015-11-12   |  

曾经被问到这样一个问题，明明修改了xib文件，但是运行之后却依然显示修改之前的xib，甚至删除了xib中的某个控件，也依然会显示出来。

试了Clean Project，删除模拟器/真机中的app，依然无用。此时我想到可能是xcode的缓存问题，打开Xcode->Preference,找到缓存地址

打开缓存文件夹，找到对应项目的文件夹删除之，重新运行，ok了。

有时候很多问题都是缓存文件搞得鬼。但是这种xib不能修改一般不会出现，遇到了两次都是在xcode版本不同的时候进行协同开发时出现的。

而且xcode的缓存文件很大，如果工程很多可以定期删除一次。
