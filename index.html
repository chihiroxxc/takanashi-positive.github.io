Mac上的使用IDEA构建Play工程

最近在实验室里决定参与一下后端开发的项目，由于实验室采用的是Scala+Play framework，所以今天配置了相关的开发环境。过程还是很简单的

安装JDK

Scala作为Java虚拟机语言(JVM Language)，与Java一样需要Java虚拟机才能编译和运行，因此开发者须首先安装Java开发工具包。官网下载安装即可。

安装IDEA

安装IDEA本来也是非常简单的，但是我在安装时选择了最新的IDEA 15社区版，导致了我后面几个小时的纠结，因为社区版并不支持play框架！！！（为什么没有人提醒我一下。。）

所以请选用收费版，如果是学生应该可以用学号申请到一年的试用期，非学生的话嘛，只要找一个注册码了。

安装插件

在IDEA上构建Play项目只需要安装Scala插件即可，我还安装了SBT，尽管前者中已经集成了SBT。

安装完毕后在/UserName中会出现.ivy2的文件夹，其中缓存了SBT下载的相关依赖，可以拷贝其他同学的该文件夹覆盖一下，节约下载时间。

Hello world

新建一个Play工程，此处SBT构建项目可能需要一些时间。

然后配置一下就可以Run了。

之后在localhost:9000即可看到激动人心的Welcome to play

修改xib无效，运行后修改没有显示的问题

曾经被问到这样一个问题，明明修改了xib文件，但是运行之后却依然显示修改之前的xib，甚至删除了xib中的某个控件，也依然会显示出来。

试了Clean Project，删除模拟器/真机中的app，依然无用。此时我想到可能是xcode的缓存问题，打开Xcode->Preference,找到缓存地址

打开缓存文件夹，找到对应项目的文件夹删除之，重新运行，ok了。

有时候很多问题都是缓存文件搞得鬼。但是这种xib不能修改一般不会出现，遇到了两次都是在xcode版本不同的时候进行协同开发时出现的。

而且xcode的缓存文件很大，如果工程很多可以定期删除一次。

Pod无法彻底删除依赖包的问题

在使用Pod的过程中添加第三方的依赖是很容易的事情，但是在某天我要对某个第三方的开源控件进行深度定制的修改时，我需要删除在pod中的引用，采用传统的方法add files 引用至工程，但是却报错，duplicated xxxx。估计就是Pod 删除的时候没有完全删除引用。
在一番寻找之后，在Target->Build Settings->Other Link Flags中找到了依然存在的第三方包引用

删除这里的引用之后pod update一下就ok了。不知道这是不是pod的bug，还是我使用有误

Hexo搭建博客时npm的permission问题

记录一下搭建hexo的过程

  1. 安装git，配置git（因为我已经安装了xcode，并且配置过git，此步跳过）
  2. 安装node.js，一开始想用源码安装的方式，但是源码怎么都下载不下来，所以去官网下载了pkg包，但是安装的npm有问题，使用 npm install 的时候会报 permission denied，后来在npm的官网找到了解决办法 Fixing npm permissions
  3. 配置Hexo，新版的deploy type 为git而非github,而且新版Hexo的git需要单独安装，其他似乎就没有要注意的地方了
