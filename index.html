求只出现一次的数的通用解法 (Single Number)

发表于 2016-09-04   |   分类于 Algorithm   |  

本文整理自
LeetCode discuss

给出一个整数数组，每个元素都出现了k(k > 1)次, 除了一个元素，出现了p(p >= 1 p % k != 0)次，找出这个元素。

我们可以用位运算来解决这个问题，首先，我们要重新思考整数在计算机中的表达方式。

本文使用java的位运算符号

  • 与运算 &
  • 或运算 |
  • 取反 ~
  • 异或 ^

先考虑1-bit的情况

首先我们假设有一个数组，里面所有的元素都是一位(1 bit)数字，只能是 0 或 1，我们想计算这个数组里有多少个1出现，如果出现的次数达到了k（上文中的k），就清零并且从头开始数。

为了记录我们从开始到现在遇到了多少个 1 ,我们需要一个计数器(counter)。假设这个counter的二进制表达式为 xm , … , x1 （从高位到低位）。我们可以总结出counter的四个性质

  • counter的初始状态是 0
  • 如果我们遇到了 0，counter应该保持不变
  • 如果遇到了 1，counter应该加 1
  • 为了能够记录 1 出现了k次，需要2m >= k

最关键的地方来了：

当我们扫描整个数组的时候，counter中的每位是如何变化的？
我们要回想出一个合适的位运算，去满足counter的第二个属性：遇到0的话counter保持不变。有两个位运算可以支持这个属性 x = x|0 和 x = x^0。

这两个运算应该使用哪个呢？我们用实际的计算来选择一下：

最初counter的所有位都是 0，如 xm = 0 , … , x1 = 0.如果我们遇到了第一个 1，我们得到xm = 0 , … , x2 = 0, x1 = 1。接下来又遇到了第二个 1，我们得到xm = 1 , … , x2 = 1, x1 = 0。 注意到了吗？x1由 1 变为了0。

如果我们使用 | 运算，在x1 = x1 | i之后，由于i是 1，x1还是会保持1，所以很显然我们应该用 ^ 操作。那么对于xm , … , x2来说呢？什么时候要改变他们的值？答：应该在右侧的位全都为 1时，这时候加一需要把左侧的一位置 1，右侧全部置 0。

所以我们得到了位运算的公式

xm = xm ^ ( xm-1 & … & x1 & i )

线段树 (Segment Tree)

发表于 2016-09-02   |   分类于 Algorithm   |  

以下是线段树在wiki线段树上的定义：

线段树（Segment Tree）是一种二叉搜索树，它将一个区间划分成一些单元区间，每个单元区间对应线段树中的一个叶结点。

对于线段树中的每一个非叶子节点[a,b]，它的左子树表示的区间为[a,(a+b)/2]，右子树表示的区间为[(a+b)/2+1,b]。因此线段树是平衡二叉树。叶节点数目为N，即整个线段区间的长度。

使用线段树可以快速的查找某一个节点在若干条线段中出现的次数，时间复杂度为O(logN)。而未优化的空间复杂度为2N，因此有时需要离散化让空间压缩。

［LeetCode］Best Time to Buy and Sell Stock 股票买卖问题

发表于 2016-08-31   |   分类于 LeetCode   |  

LeetCode上有五道股票买卖的题目，都是动态规划问题，从Easy到Hard，下面总结一下这五道题目，写出 step by step 的思路。

题目的内容都是相同的

用一个数组表示股票每天的价格，如[1,0,5,7,2,6],在不同条件的约束下，如何取得最大的收益。（不可以在同一天进行买入和卖出）

I. 只可以买卖一次

这是最简单的问题，只需要在最低的时候买入，在最高的时候卖出就行了。但是显然要满足买入时间在卖出时间之前。所以我们可以用min记录在今天之前出现的最小价格，max记录在今天能够取得的最大值。

II. 可以买卖多次

这道题目在之前题目的基础上，不限制卖买的次数，所以我们获得的最大收益是个累计值，并不是一次的值。

动态规划题目的特征是重叠子问题和最优子结构

分析股票买卖问题，其实具有很明显的DP特征，如果能够买卖多次的话，当天的最大收益就是当天持有股票和不持有股票时两种收益的最大值。

持有股票时的收益与前一天持有股票的收益和今天买入股票的收益有关

不持有股票的收益与前一天不持有股票的收益和今天卖出股票的收益有关

很容易写出状态转移方程，dp[i][0]表示在第i天不持有股票的收益，dp[i][1]表示在第i天持有股票的收益。

dp[i][0] = Math.max(dp[i-1][0],dp[i-1][1]+prices[i]);
dp[i][1] = Math.max(dp[i-1][1],dp[i-1][0]-prices[i]);

当面对一个排序数组时，可能会被问到二分搜索（Binary Search）这个算法。但如果一个排序好的数组被旋转了呢？

旋转指的是在某个点处划分数组为两部分，然后这两部分内容交换。
如数组[0, 1, 2, 4, 5, 6, 7] 在4前面划分，交换左右两部分，成为新的旋转排序数组[4, 5, 6, 7, 0, 1, 2]

那么面对这样的一个数组，还有办法进行二分查找吗？

首先我们来看一个比查找简单的问题，如何找到一个旋转数组的最小值（该数组里没有重复元素）。

找到旋转数组的最小值

如果遍历数组肯定可以找出最小值，但这样浪费了数组被部分排序的属性。

如何利用这部分排序的属性呢？

我们先按照二分法的思路,取到中间的值nums[mid]，如果这个最小值在左边的话，那么从mid到high肯定是排序好的，如果这个最小值在右边的话，那么从mid到low肯定是排序好的。我们可以根据nums[mid]和nums[high]的大小关系来判断最小值在左侧还是右侧。

这样的时间复杂度是O(NlogN)。

需要注意的是，再写二分查找时，循环的条件和low,high的更新值很重要。

(2) Given nums = [1, 3, 2, 2, 3, 1], one possible answer is [2, 3, 1, 3, 1, 2].

Note:
You may assume all input has valid answer.

Follow Up:
Can you do it in O(n) time and/or in-place with O(1) extra space?

这道题目要求我们把一个无序数组排成“小-大-小-大”这样间隔交错的形式。

O(NlogN)时间复杂度O(N)空间复杂度的方法

先排序再交错排列是最简单的方法也是效率最低的方法。

面试中经常会问到关于java中String的问题，String是个非常常用的类，关于String问题不容忽视

java中的String和其他语言一样，也是字符序列，但是对java来说，String更像是一个字符序列(char sequence)的辅助类。String用以下几个变量来维护字符序列

存储字符序列的数组：

private final char value[];

value数组对应的起始位置的偏移量：

private final int offset;

value数组对应的字符个数：

private final int count;

偏移量不一定为0，对应的字符个数也不一定为value.length

为什么String类是不可变的(Immutable)

  • 第一个原因是为了提高性能。 应用程序往往包含了非常多的String, 标签，消息，设置，等等。JVM的设计者们想出了一个String 常量池的概念，常量池也就是一系列string的集合（大部分的值都是唯一的），常量池可以提供String的复用。如果一个String被创建了20次，那么在常量池中将会只有一个实例存在。

  • 第二个原因是为了安全起见，String类不允许其他类继承。
    因为String类对于Java来说是一个非常常用而且重要的类，随意继承可能会引起意想不到的问题。

［笔记］Java 内存模型

发表于 2016-08-20   |   分类于 笔记   |  

最近看了一篇很好的关于 Java 内存模型的文章，所以记下笔记以便日后复习。

Java Memory Model

Java 的内存模型决定了 Java Virtual Machine 是如何和计算机的内存协作的。JVM是一个包含了整个机器的模型，所以自然也包括了内存模型，即我们所说的Java内存模型（Java Memory Model）。

如果你想正确的使用Java的多线程并发机制，你就必须理解JMM，因为JMM指定了多线程是如何协作的，比如一个线程是怎样看到一个共享的变量被其他线程修改，或者是如何在必要的时候同步共享的变量。

JMM

JMM在JVM内部将内存分为了线程栈和堆两部分。如下图。

每个运行在JVM中的线程都有一个自己的线程栈，其中包括了调用过的方法的信息。我们可以把它叫做“调用堆栈”，当线程执行它的代码，调用堆栈就会被改变。

栈里面还保存了方法中所用到的所有局部变量。一个线程只能访问自己的栈，而栈中的变量也只能为该线程所见。就算两个线程执行了同一段代码，也会在各自的线程栈中访问所需要的局部变量。

LeetCode No.146 LRU Cache

发表于 2016-08-19   |   分类于 LeetCode   |  

LRU是 Least Recent Used 的缩写。即最近最少使用，是一种页面置换算法。

Design and implement a data structure for Least Recently Used (LRU) cache. It should support the following operations: get and set.

get(key) - Get the value (will always be positive) of the key if the key exists in the cache, otherwise return -1.

set(key, value) - Set or insert the value if the key is not already present. When the cache reached its capacity, it should invalidate the least recently used item before inserting a new item.

分析一下题目，要求实现get(key)和set(key, value)，肯定要用到HashMap来实现键值对的存储。但是系统提供的HashMap无法满足在即将超过capacity的时候从HashMap中删除最近最少使用的数据。

那么什么是最近最少使用呢？考虑下面的操作：

Operation Cache(capacity:3)
set(1, val) 1
set(2, val) 2 ,1
set(3, val) 3 ,2, 1
get(2) 2 ,3, 1
get(1) 1 ,2, 3
set(4, val) 4, 1 ,2

可以看出数据除了有key-value的性质，还像链表一样被连接起来，每次被get的数据被移动到链表头，如果当前已经达到了capacity，再次set将会删除链表的最后一个数据，并将新的数据插入到链表头。
