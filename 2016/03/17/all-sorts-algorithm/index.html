【总结篇】所有常见的排序算法

  • Algorithm

虽然排序是入门的算法，但是也是很有必要总结一下的，距离第一次学习排序算法已经快四年了。而Java提供了Aarrys.sort()方法让我基本上没有再手写过排序算法。大一学习数据结构，接触到了冒泡排序，当时让我手写冒泡还很费劲。快排愣是看了半天都没懂……

出现在这片总结里的排序算法有一下几种：

  • 冒泡排序
  • 插入排序
  • 归并排序
  • 堆排序
  • 快速排序
  • （其他待续）

冒泡排序

冒泡排序的基本思想是，通过一次遍历，找出一个当前最大（最小）值，排在最后（最前）。是一种原地排序的方法，其时间复杂度是O(n^2)。

public void bubble_sort(int[] A) {
    for (int i = 0; i < A.length; i++) {
        for (int j = 1; j < A.length - i; j++) {
            if (A[j] < A[j - 1]) {
                //swap
                int temp = A[j];
                A[j] = A[j-1];
                A[j-1] = temp;
            }
        }
    }
}

插入排序

插入排序是把排序的过程想象为打扑克时整理手牌的过程，假设手中的牌已经是排序好的，那么将一张新的手牌插入已排序好的手牌，这过程就是插入排序。插入排序的时间复杂度也是O(n^2)。

public void insert_sort(int[] A) {
    for (int i = 1; i < A.length; i++) {
        int j = i;
        while (j > 0 && A[j] < A[j - 1]) {
            int temp = A[j - 1];
            A[j - 1] = A[j];
            A[j] = temp;
            j--;
        }
    }
}

归并排序

这是一个时间复杂度为O(nlgn)的排序算法。其本质思想是分治法：Divide and conquer。

将一个大问题分解为最小的问题，再将其合并为大问题的解。在归并排序中，最小的问题即两个数的排序 a>b or a<b or a==b。解决了最小的问题后，下一步是将最小的问题合并，即：合并两个已排序的数组。最终得到一个排序好的完整数组。通过递归可以顺利的解决该问题~

public void merge_sort(int[] A, int p, int r) {
    if (p < r) {
        int q = (p + r) / 2;
        merge_sort(A, p, q);
        merge_sort(A, q + 1, r);
        merge(A, p, q, r);
    }
}

public void merge(int[] A, int p, int q, int r) {
    int n1 = q - p + 1;
    int n2 = r - q;
    int[] L = Arrays.copyOfRange(A, p, q + 1);
    int[] R = Arrays.copyOfRange(A, q + 1, r + 1);
    int i = 0, j = 0, k = p;
    while (i < n1 && j < n2) {
        if (L[i] < R[j]) {
            A[k++] = L[i];
            i++;
        } else {
            A[k++] = R[j];
            j++;
        }
    }
    while (i < n1) {
        A[k++] = L[i];
        i++;
    }
    while (j < n2) {
        A[k++] = R[j];
        j++;
    }
}

第一次调用的话就是：merge_sort(A,0,A.length-1)

堆排序

堆可以被视为一棵完全二叉树，用数组存储，故得知某个节点的下标后，很容易得知其父母，左儿子和有儿子的节点下标。

// parent: i/2
// left child: i*2
// right child: i*2+1

对于堆来说，当堆建成之后，每次获取最大值（对于大顶堆而言）的时间复杂度都是O(1)，因为（大顶堆）可以保证永远都在第一个下标。

对于堆排序来说，其步骤大概是这样的：首先将无序数组构造成一个大顶堆，每次拿去顶点的最大值，放在队尾，继续将前n-1个数构造大顶堆。

public void heap_sort(int[] A) {
    build_max_heap(A);
    int heap_size = A.length;
    for (int i = A.length - 1; i > 0; i--) {
        int temp = A[i];
        A[i] = A[0];
        A[0] = temp;
        heap_size--;
        max_heapify(A, 0, heap_size);
    }
}

public void build_max_heap(int[] A) {
    for (int i = A.length / 2; i >= 0; i--) {
        max_heapify(A, i, A.length);
    }
}

public void max_heapify(int[] A, int i, int heap_size) {
    if (i >= 0) {
        int left = ((i + 1) << 1) - 1;
        int right = left + 1;
        int largest = i;
        if (left < heap_size && A[i] < A[left]) {
            largest = left;
        }
        if (right < heap_size && A[largest] < A[right]) {
            largest = right;
        }
        //swap
        if (i != largest) {
            int temp = A[i];
            A[i] = A[largest];
            A[largest] = temp;
            max_heapify(A, largest, heap_size);
        }
    }
}

max_heapify是对最大堆进行操作的子程序，其目的是保持最大堆的性质。堆排序的时间代价为O(nlgn),其中调用build_max_heap的时间为O(n),n-1次max_heapify调用中每次的时间代价为O(lgn)

快速排序

快速排序是用于排序的最佳实用选择，因为其平均性能相当好（虽然其最坏的运行时间为O(n^2)）。
快速排序也是基于分治法的思想，对于一个无序的数组，将最后一个元素作为主元，通过partition将其就地重排，排序结果是主元位于数组中间，主元左边的元素都小于等于主元，右边的元素都大于等于主元。然后对主元左边和右边的子数组分别再次调用partition。直到子数组无可划分，排序停止。

public void quick_sort(int[] A, int p, int r) {
    if (p < r) {
        int i = p - 1;
        int x = A[r];
        for (int j = p; j < r; j++) {
            if (A[j] <= x) {
                i++;
                int temp = A[i];
                A[i] = A[j];
                A[j] = temp;
            }
        }
        int temp = A[i + 1];
        A[i + 1] = x;
        A[r] = temp;
        quick_sort(A, p, i);
        quick_sort(A, i + 1, r);
    }
}
