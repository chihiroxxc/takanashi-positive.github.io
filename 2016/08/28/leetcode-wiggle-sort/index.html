LeetCode No.324 Wiggle Sort II

发表于 2016-08-28   |   分类于 LeetCode   |  

Given an unsorted array nums, reorder it such that nums[0] < nums[1] > nums[2] < nums[3]….

Example:
(1) Given nums = [1, 5, 1, 1, 6, 4], one possible answer is [1, 4, 1, 5, 1, 6].
(2) Given nums = [1, 3, 2, 2, 3, 1], one possible answer is [2, 3, 1, 3, 1, 2].

Note:
You may assume all input has valid answer.

Follow Up:
Can you do it in O(n) time and/or in-place with O(1) extra space?

这道题目要求我们把一个无序数组排成“小-大-小-大”这样间隔交错的形式。

O(NlogN)时间复杂度O(N)空间复杂度的方法

先排序再交错排列是最简单的方法也是效率最低的方法。

举个例子[1,3,4,5,1,6] 先排列成[1,1,3,4,5,6]。数组可以被看做分成了两部分——较小部分和较大部分。将较小部分插入下标为偶数的槽，得到

[1, ,1, ,3, ,]

将较大部分插入下标为奇数的槽，得到[1,4,1,5,3,6]

这样一定是可以的吗？我们只能保证较大部分大于等于较小部分，但却不能保证一定大于较小部分，因为有可能有重复的数字存在。

比如[1,2,5,5,6,7]。第一个5属于较小部分，第二个5属于较大部分。如果交错排列不得当的话，很有可能将两个5排在一起。但是上面说的方法可以避免这种情况，为什么呢？我们下面把数组抽象成Small, Medium, Large，三个部分（S < M < L）。有可能出现相同排列的地方就是M了。

这样可以清楚的看到，M被分散在了数组两边，不会相遇，那如果M的个数再多一点呢？

  • 数组长度为n时，M最多有floor(n/2)个的时候，这种算法不会出错
  • M>floor(n/2)时，不存在一种Wiggle Sort的解，因为要想得出一个Wiggle Sort的解，数组里至少要有和M一样多个小于M的数，这与假设相矛盾

证明了结论后我们可以写代码了。这里有个很有用的函数，可以将当前下标映射到Wiggle Sort数组的下标。

O(N)时间复杂度 in-place

上面的解法可以从时间复杂度和空间复杂度两方面进行优化。

时间复杂度

由于我们最终的目的只是把数组分成两部分，至于分好的两部分，不需要是有序的。所以我们可以采用partition的思想，将数组分成大小两部分。(这里我直接试使用了另一道leetcode题目find Kth Largest Number的代码，原理是一样的)

空间复杂度

降低空间复杂度，最好能做到原地交换。这是我们的目标。

但是由于我们放弃了排序整个数组，所以我们无法得到在上个解法中的S<M<L的有序数列，我们的得到的数组很有可能是[M,S,M,S,S,L,M,L,M]这种形式，所以按照上面解法的交错方式，是有可能将两个M排列在一起的。

怎么避免这种情况的发生呢？

  • 首先我们通过findKthLargest得到M的值
  • 对于小于M的值，放入最后一个偶数槽中
  • 对于大于M的值，放入第一个奇数槽中
  • 等于M的值，放入其余的槽中
