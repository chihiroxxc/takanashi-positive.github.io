［笔记］面试中常见的java String相关问题

发表于 2016-08-22   |   分类于 笔记   |  

面试中经常会问到关于java中String的问题，String是个非常常用的类，关于String问题不容忽视

java中的String和其他语言一样，也是字符序列，但是对java来说，String更像是一个字符序列(char sequence)的辅助类。String用以下几个变量来维护字符序列

存储字符序列的数组：

private final char value[];

value数组对应的起始位置的偏移量：

private final int offset;

value数组对应的字符个数：

private final int count;

偏移量不一定为0，对应的字符个数也不一定为value.length

为什么String类是不可变的(Immutable)

  • 第一个原因是为了提高性能。 应用程序往往包含了非常多的String, 标签，消息，设置，等等。JVM的设计者们想出了一个String 常量池的概念，常量池也就是一系列string的集合（大部分的值都是唯一的），常量池可以提供String的复用。如果一个String被创建了20次，那么在常量池中将会只有一个实例存在。

  • 第二个原因是为了安全起见，String类不允许其他类继承。
    因为String类对于Java来说是一个非常常用而且重要的类，随意继承可能会引起意想不到的问题。

字符串常量池(String pool)

String pool 是一个堆之外的存储区域，在那里存储的全都是字符常量，这些常量会被字符串变量引用。

在java中，我们有几种方式来创建一个String

直接赋值

String str = "abc";

这种创建方式将会调用JVM去确认在常量池中是否已经有"abc"存在，如果存在的话，只需要把引用赋给str即可，如果不存在，一个新的字符串常量将会被创建。

使用关键字new来创建

String str = new String("abc");

这种方式将会最终在内存中创建两个对象，一个是常量池中的"abc"，另一个是堆中的和"abc"拥有一致序列的字符串对象，str将会指向这个对象。

java doc中建议：除非真的需要copy一个String，否则不需要使用new关键字来创建对象。

可能会造成的内存泄漏

当使用substring时，会创建一个新的String对象，但是values[]不会改变，只是会更新offset和count值。
